System to automatically generate release notes from semantic commit messages

## Commit Message Conventions

```
<type>(<scope>): <description>

<body>

<footer>
```

Required fields are:

- `<type>`: the type of change
- `<description>`: brief summary of changes (50 chars max)

Optional fields are:

- `(<scope>)`: component or module affected, eg: auth, api, ui, header, etc
- `<body>`: detailed explanation
- `<footer>`: additional metadata (breaking changes, issue links)

## Breaking Changes

- Add `!` after type: `feat!: redesign user auth`
- or use footer: `BREAKING CHANGE: redesign user auth`

## Examples

```bash
# Feature with scope
feat(auth): add oauth2

# Bug fix with scope and link to issue in footer
fix(api): resolve memory issue in user server

Fixes #123

# Breaking changes
feat(api)!: redesign user endpoint
```

## Versioning logic

- Major: Contains breaking change (`!` or `BREAKING CHANGE`)
- Minor: Contains new features (`feat`)
- Patch: Contains: `fix`, `perf`, `revert` (if applicable)

## Libraries

- [Semantic Release](https://github.com/semantic-release/semantic-release) to generate release note pages in Github
- [Husky](https://github.com/typicode/husky) to enforce Semantic Release commit messages

## Github Workflow

Created a Workflow file that creates release notes when there are pushes to a `release` branch:

```yml
name: Generate Release Notes
on:
  push:
    branches:
      - release

permissions:
  contents: write
  issues: write
  pull-requests: write
  id-token: write

jobs:
  release:
    name: Generate Release Notes
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3
        with:
          fetch-depth: 0
          persist-credentials: false

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: "24"

      - name: Install dependencies
        run: npm ci

      - name: Release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: npx semantic-release
```
